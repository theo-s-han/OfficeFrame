## 현재 구현 단계

- 홈 허브와 간트 에디터 MVP가 구현되어 있다.
- 간트 에디터는 타입 선택, Frappe Gantt preview, task CRUD, validation, PNG export를 제공한다.
- resource, critical path, baseline, 저장/협업 기능은 아직 포함하지 않는다.

## 1. 홈 허브

### 목적

- 사용자가 현재 제공되는 도구와 앞으로 추가될 도구를 빠르게 이해하고 선택하게 한다.

### 주요 사용자 행동

- 사용자는 간트 차트 카드를 눌러 간트 에디터로 진입한다.
- 사용자는 placeholder 도구가 아직 준비 중임을 이해한다.

### 입력값

- 없음

### 출력값

- 도구 카드 그리드
- 활성 도구와 준비 중 도구의 구분

### 레이아웃

- 상단: 제품명과 현재 단계 표시
- 본문: 짧은 설명과 도구 카드 그리드
- 카드: 아이콘, 도구명, 설명, 상태, 시작 버튼

### 상태

- 초기 상태: 간트 차트는 활성, 마인드맵/조직도/플로우차트/타임라인은 준비 중
- 오류 상태: 없음

### 액션

- `시작하기`: `/gantt`로 이동
- `준비중`: 비활성 표시

### 성공 기준

- 첫 화면에서 간트 차트가 유일한 활성 진입점임을 알 수 있다.
- placeholder 도구가 구현된 기능처럼 보이지 않는다.

## 2. 간트 에디터

### 목적

- 간트 데이터를 입력/수정하고 preview를 확인한 뒤 문서용 PNG로 저장한다.

### 주요 사용자 행동

- 사용자는 하단 row editor에서 task를 추가/수정/삭제한다.
- 사용자는 상단에서 간트 타입을 선택한다.
- 사용자는 날짜 단위를 1일, 주, 월, 분기 중 선택하고 표시 시작/종료 범위를 정한다.
- 사용자는 우측 Frappe Gantt preview에서 막대를 드래그해 일정을 조정한다.
- 사용자는 preview의 일정 막대를 클릭해 해당 입력 행으로 이동하고 바로 편집한다.
- 사용자는 기본 일정표 task의 색상 선택 창에서 팔레트, 컬러 피커, HEX 입력으로 막대 색상을 정한다.
- 사용자는 현재 preview를 PNG로 다운로드한다.

### 입력값

- Task
- Color
- Start
- End
- Progress
- Type
- Phase
- Owner
- Status
- Baseline Start
- Baseline End
- Dependencies
- Background Template
- Date unit
- Timeline Start
- Timeline End

### 출력값

- 상단 toolbar
- 간트 타입 selector
- 상단 Frappe Gantt preview
- 하단 row editor
- validation 메시지
- PNG 파일

### 레이아웃

- 상단: 간트 타입 selector와 action toolbar
- toolbar 하단: 날짜 단위와 표시 범위
- 본문 상단: 큰 preview 영역, 넘치는 일정은 내부 스크롤
- 본문 하단: 행 단위 task 입력 영역

### 상태

- 초기 상태: sample tasks 표시
- 빈 상태: 작업 추가 안내 표시
- 오류 상태: 필드 또는 행 단위 validation 메시지 표시
- 성공 상태: PNG 다운로드 완료 메시지 표시

### 액션

- 예시 데이터
- 전체 초기화
- 기본 일정표/로드맵형/마일스톤형/진행률 추적형/WBS형 타입 전환
- 1일/주/월/분기 날짜 단위 전환
- 표시 시작/종료 범위 변경
- 기본 일정표 task 색상 변경
- 기본 일정표 task 색상 선택 창 열기
- 기본 일정표 배경 템플릿 변경
- 오늘로 이동
- PNG 다운로드
- 작업 추가/삭제
- preview task 클릭 기반 입력 행 선택
- 차트 drag/resize 기반 일정 수정
- 차트 클릭 선택은 preview 재생성 없이 현재 chart instance를 유지

### 예외 처리

- 빈 Task, 잘못된 날짜, End < Start, Progress 범위 오류를 표시한다.
- 차트 drag에서 유효하지 않은 날짜 값이 들어오면 이전 task 값을 유지하고 debug mode에 rollback 이벤트를 남긴다.
- 오류가 있으면 PNG 다운로드를 막는다.

### 성공 기준

- 데스크톱에서 preview가 먼저 크게 보이고 입력 행은 아래에서 안정적으로 편집된다.
- task 입력 영역은 내부 스크롤 없이 행 개수만큼 아래로 확장된다.
- preview는 화면 폭 밖으로 새지 않고 내부 스크롤로 넘친 기간을 확인할 수 있다.
- preview 내부 세로 스크롤이 끝나면 페이지 스크롤로 자연스럽게 이어진다.
- preview task를 클릭하면 해당 입력 행이 선택되고 편집 포커스를 받는다.
- preview task 클릭만으로 전체 차트가 다시 그려지지 않는다.
- 드래그 결과가 유효하지 않으면 화면 데이터가 사라지지 않고 이전 일정으로 되돌아간다.
- task 추가/수정/삭제가 preview에 반영된다.
- Frappe Gantt preview가 렌더링된다.
- PNG export가 동작한다.
- 타입을 바꾸면 입력 필드와 preview 스타일이 바뀐다.
- 타입별 입력은 기본 일정표, 로드맵형, 마일스톤형, 진행률 추적형, WBS/단계형의 실제 사용 목적에 맞게 달라진다.
- 날짜 단위를 바꾸면 task와 표시 범위 입력 UI가 함께 바뀐다.
- 주 단위 preview는 월 헤더 아래에 월 내부 주차를 1주, 2주, 3주처럼 표시한다.
- 기본 일정표에서 task 색상 버튼을 누르면 색상 선택 창이 열리고, 프리셋/직접 선택/HEX 입력으로 고른 색상이 preview와 PNG 대상에 반영된다.
- 기본 일정표에서 배경 템플릿을 바꾸면 preview와 PNG 대상에 반영된다.

## 공통 UX 원칙

- 실무형 B2B 톤을 유지한다.
- 데스크톱 사용성을 우선한다.
- 색보다 구조로 이해되게 한다.
- 빈 화면보다 예시 데이터나 안내를 우선 검토한다.
- 위험한 액션은 MVP 단계에서 확인 단계를 둔다.
