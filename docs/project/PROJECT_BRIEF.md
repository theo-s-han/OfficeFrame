# PROJECT_BRIEF.md

## 프로젝트 정의

- 제품명: Office Tool
- 한 줄 설명: 오피스 문서에 바로 붙여넣기 좋은 구조형 시각화 이미지를 만드는 웹앱
- 해결하려는 문제: 실무 문서에 필요한 간트 차트, 조직도, 플로우차트 같은 시각화를 엑셀/문서 도구만으로 빠르게 만들기 어렵다.

## 타겟 사용자

- 주요 사용자: 기획자, PM, 개발 리드, 운영 담당자 등 실무 직장인
- 사용 맥락: 보고서, 기획서, 일정 공유 문서, PPT에 넣을 시각화 이미지가 필요할 때
- 가장 중요한 사용자 행동: 구조화된 데이터를 입력하고 문서용 이미지로 빠르게 저장한다.

## 성공 기준

- 제품 성공 지표: 첫 사용자가 학습 비용 없이 기본 결과물을 만들 수 있다.
- MVP 성공 판단 기준: 간트 데이터를 입력/수정하고 preview를 확인한 뒤 PNG로 저장할 수 있다.
- 실행 골격 성공 판단 기준: 홈 허브와 간트 에디터 shell이 실행되고 lint/typecheck/test/build가 통과한다.

## MVP 범위

- 반드시 만드는 것: 홈 허브, 기본 간트 차트 도구, 좌측 편집 패널, 우측 preview, PNG export
- 현재 구현된 것: Next.js 실행 환경, 홈 허브, 간트 에디터, Frappe Gantt preview, task CRUD, validation, PNG export, debug mode
- 제외하는 것: AI/LLM 기능, 백엔드, 로그인, 협업, 클라우드 저장, 결제, 고급 drag/drop, 고급 간트 기능

## 핵심 시나리오

1. 사용자는 홈 허브에서 간트 차트 도구를 선택한다.
2. 간트 에디터에서 작업명, 시작일, 종료일, 진행률을 입력하거나 수정한다.
3. 우측 preview를 확인하고 문서에 붙여넣을 PNG 결과물을 얻는다.

## 제약사항

- 기술 제약: 프론트엔드 중심, 단일 repo, plugin 확장 가능 구조
- 비용 제약: 실행 시 지속 비용이 발생하는 외부 LLM/API 기능 없음
- 제품 제약: 데스크톱 우선, 실무형 B2B UI
- 범위 제약: 고급 간트 기능은 별도 승인 전까지 구현하지 않는다.

## 가정

- 현재 맞다고 보는 전제: 간트 차트는 첫 MVP 도구로 실무 가치와 구현 가능성이 높다.
- 아직 검증되지 않은 전제: Frappe Gantt 기반 PNG export 품질이 실제 PPT/문서 사용에서 충분하다.

## 오픈 이슈

- PNG export 구현 라이브러리 선택
- Frappe Gantt의 상업 사용/라이선스 최종 확인
- 배포 도메인과 제품명 확정
